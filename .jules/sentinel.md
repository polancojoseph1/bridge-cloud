## 2023-10-24 - SSRF FQDN Bypass
**Vulnerability:** The `isForbiddenHostname` SSRF filter can be bypassed by appending a trailing dot (`.`) to the hostname (e.g., `127.0.0.1.`, `localhost.`), known as a Fully Qualified Domain Name (FQDN). It can also be bypassed using URL encoded dots (`%2e`). Node.js's `URL` parser preserves the trailing dot in `url.hostname`, which bypasses strict exact matches or regexes in the filter. When the request is passed to `fetch()`, the underlying network stack normalizes the trailing dot and successfully requests the internal resource.
**Learning:** Hostnames parsed from URLs must have trailing dots trimmed (and ideally decoded) before validation, because URL parsers and network request libraries (like `fetch` and DNS lookups) treat `127.0.0.1.` identically to `127.0.0.1`, while simplistic string filters do not.
**Prevention:** Always normalize the hostname by explicitly stripping any trailing dots (e.g., `hn.replace(/\.+$/, '')`) and ensuring the string is fully decoded before passing it to any blocklist checks.
