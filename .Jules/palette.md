## 2025-04-21 - Fix Keyboard Accessibility in ServerSwitcherPopover
**Learning:** Native `<button>` elements implicitly act as focusable and interactive elements via the 'Tab' and 'Enter'/'Space' keys. However, when these elements are used in a custom popover or dropdown, standard browser behaviors (like scrolling) might interfere, or the user might expect standard popover navigation (like arrow keys, or consistent styling when focused). Applying `focus-visible:outline-none` removes the browser's default focus ring, which is crucial for custom styling. But if you forget to add a custom focus ring (like `focus-visible:ring-2`), you completely break keyboard accessibility for sighted users navigating via Tab.
**Action:** When overriding the default focus outline (`focus-visible:outline-none`) on interactive elements, always provide an equivalent custom focus indicator (e.g., `focus-visible:ring-2 focus-visible:ring-inset focus-visible:ring-[#6c8cff]`). This ensures the element remains clearly distinguishable when receiving keyboard focus.
